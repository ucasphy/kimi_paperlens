# 速览模式 — Speed Read

## 目标

5 分钟内让读者消化一篇论文的核心，快速判断是否值得深读。

## 输出结构

在对话中直接回复，使用 markdown 格式。同时保存为 `paper-notes/<name>/speed-read.md`。

---

### 第一部分：基本信息卡片

用引用块格式输出：

```markdown
> **论文**：<英文标题>
> **中文标题**：<中文翻译>
> **作者**：<前三位作者 等（机构）>
> **发表**：<会议/期刊名称 + 年份>（如未发表标注 arXiv + 日期）
> **一句话总结**：<用一句话说清楚这篇论文做了什么、为了什么、效果怎样>
> **阅读难度**：<⭐ 到 ⭐⭐⭐⭐⭐>
> **领域定位**：<开创性 / 改进型 / 应用型 / 综述型>
> **前置知识**：<理解这篇论文需要的 2-3 个前置概念>
```

**难度等级说明**：
- ⭐ 入门级：无需专业背景
- ⭐⭐ 基础级：了解深度学习基础
- ⭐⭐⭐ 进阶级：熟悉相关领域
- ⭐⭐⭐⭐ 专业级：需要较深的领域知识
- ⭐⭐⭐⭐⭐ 专家级：涉及复杂数学或前沿研究

**领域定位说明**：
- 开创性：提出了全新的问题/方法/范式
- 改进型：在已有方法上做出显著改进
- 应用型：将已有方法应用到新场景
- 综述型：系统梳理某个领域的进展

**一句话总结要求**：
- 必须包含：做了什么 + 为了什么 + 效果/结论
- 不超过 50 个中文字
- 避免泛泛而谈，要有具体技术点

---

### 第二部分：核心创新

```markdown
## 核心创新

<2-3 句话概括，不超过 100 字。必须回答：这篇论文最独特的贡献是什么？与之前的工作相比，最关键的区别是什么？>
```

**要求**：
- 不是罗列贡献，而是提炼最核心的 1-2 个创新点
- 用「不同于...，本文...」的对比句式来突出新意
- 避免"提出了一种新方法"这类废话，要说清楚新在哪里

---

### 第三部分：主要方法

```markdown
## 主要方法

**整体思路**：<一句话概括方法论的核心逻辑>

**关键技术点**：
- <技术点1>：<一句话说明>
- <技术点2>：<一句话说明>
- <技术点3>：<一句话说明>
```

**要求**：
- 整体思路用一句话，让读者知道"大致怎么做的"
- 关键技术点 3-5 个，每个用一句话
- 不展开细节，不放公式，留给学习模式
- 如果有架构图可以引用：`![架构图](images/fig_xxx.png)`

---

### 第四部分：实验结果

```markdown
## 实验结果

**核心数字**：
- <最重要的性能指标>：<数字>（对比基线提升 X%）
- <次要指标>：<数字>

**主要发现**：
1. <发现1：一句话>
2. <发现2：一句话>
3. <发现3：一句话>（如有）

**局限性**：<论文承认的或你观察到的 1-2 个局限>
```

**要求**：
- 核心数字要有对比基准，不是孤立的数字
- 主要发现提炼结论而非罗列数据
- 局限性要客观，不夸大也不忽视

---

### 第五部分：总结与 QA

```markdown
## 总结与 QA

**贡献总结**：
1. <贡献1>
2. <贡献2>
3. <贡献3>（如有）

**Q1：<一个关于"为什么"的问题>**
<简洁回答>

**Q2：<一个关于"怎么做"的问题>**
<简洁回答>

**Q3：<一个关于"和其他方法相比"的问题>**
<简洁回答>
```

**QA 设计原则**：
- 三个问题覆盖：动机（为什么要做）、方法（核心怎么做的）、定位（和别人比怎么样）
- 回答简洁，每个 2-3 句话
- 这些 QA 应该是读者最可能问的问题

---

## 输出要求

- 全部使用中文
- 总长度控制在 300-500 字（不含 QA）
- 一次性输出，不需要和用户交互
- 输出完成后提示：「速览完成。想深入了解可以切换到学习模式，想准备汇报可以切换到展示模式。」
